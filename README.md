# DermoScan

DermoScan is a Flask-based skin infection detection web app that lets users upload skin images, run an AI-based prediction, view scan history, and manage an account with Supabase-backed storage.

## Features
- User registration and login with a custom Supabase users table
- Skin image upload and prediction
- Scan history with image thumbnails
- PDF report generation
- Premium/upgrade flow
- Admin panel

## Tech stack
- Python 3.11
- Flask
- Supabase
- Jinja templates
- ReportLab
- TensorFlow / Keras model support

## Project structure
- app.py: main Flask application
- database.py: Supabase helpers for users and scan storage
- predictor.py: prediction and model loading logic
- report.py: PDF report generation
- templates/: HTML templates
- static/: uploaded assets and static files
- api/: Vercel-compatible serverless entry point

## Local setup
1. Create and activate a virtual environment
   ```bash
   python -m venv .venv
   .venv\Scripts\activate
   ```
2. Install dependencies
   ```bash
   pip install -r requirements.txt
   ```
3. Set environment variables
   ```bash
   set SUPABASE_URL=your_supabase_url
   set SUPABASE_KEY=your_supabase_key
   set SUPABASE_SERVICE_ROLE_KEY=your_service_role_key
   set SECRET_KEY=change-me
   ```
4. Run the app
   ```bash
   python app.py
   ```

## Supabase setup
Create the following tables in Supabase:

### users
```sql
create table public.users (
  id text primary key,
  first_name text,
  last_name text,
  username text unique,
  password_hash text,
  is_premium boolean default false,
  is_admin boolean default false,
  created_at timestamptz default now()
);
```

### scan_history
```sql
create table public.scan_history (
  id bigint generated by default as identity primary key,
  user_id text not null,
  image_filename text,
  image_bytes text,
  predicted_class text,
  confidence double precision,
  all_scores text,
  created_at timestamptz default now()
);
```

### feedback
```sql
create table public.feedback (
  id bigint generated by default as identity primary key,
  user_id text not null,
  message text,
  rating integer,
  created_at timestamptz default now()
);
```

## Deployment options

### Railway deployment
This project is configured for Railway. Deployment files are included in [Procfile](Procfile) and [railway.json](railway.json).

#### Required environment variables for Railway
- SECRET_KEY
- SITE_URL (e.g. `https://your-app-name.up.railway.app`)
- AUTH_CONFIRM_PATH
- SUPABASE_URL
- SUPABASE_KEY
- SUPABASE_SERVICE_ROLE_KEY

#### Railway deploy steps
1. Push the repo to GitHub.
2. Go to [railway.app](https://railway.app) and create a new project.
3. Select **Deploy from GitHub repo** and connect the repository.
4. Railway will auto-detect the Nixpacks builder and use the start command from `railway.json`:
   ```bash
   gunicorn app:app
   ```
5. Open the **Variables** tab and add the environment variables above.
6. Deploy — Railway will build and serve the app automatically.

> The `railway.json` file sets the builder to Nixpacks, the start command to `gunicorn app:app`, and configures automatic restarts on failure.

## Notes
- For serverless hosting, keep the app lightweight and ensure the model fallback path is available.
- The app uses Supabase for persistence instead of a local database.
