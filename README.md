# DermoScan

DermoScan is a Flask-based skin infection detection web app that lets users upload skin images, run an AI-based prediction, view scan history, and manage an account with Supabase-backed storage.

## Features
- User registration and login with a custom Supabase users table
- Skin image upload and prediction
- Scan history with image thumbnails
- PDF report generation
- Premium/upgrade flow
- Admin panel

## Tech stack
- Python 3.11
- Flask
- Supabase
- Jinja templates
- ReportLab
- TensorFlow / Keras model support

## Project structure
- app.py: main Flask application
- database.py: Supabase helpers for users and scan storage
- predictor.py: prediction and model loading logic
- report.py: PDF report generation
- templates/: HTML templates
- static/: uploaded assets and static files
- api/: Vercel-compatible serverless entry point

## Local setup
1. Create and activate a virtual environment
   ```bash
   python -m venv .venv
   .venv\Scripts\activate
   ```
2. Install dependencies
   ```bash
   pip install -r requirements.txt
   ```
3. Set environment variables
   ```bash
   set SUPABASE_URL=your_supabase_url
   set SUPABASE_KEY=your_supabase_key
   set SUPABASE_SERVICE_ROLE_KEY=your_service_role_key
   set SECRET_KEY=change-me
   ```
4. Run the app
   ```bash
   python app.py
   ```

## Supabase setup
Create the following tables in Supabase:

### users
```sql
create table public.users (
  id text primary key,
  first_name text,
  last_name text,
  username text unique,
  password_hash text,
  is_premium boolean default false,
  is_admin boolean default false,
  created_at timestamptz default now()
);
```

### scan_history
```sql
create table public.scan_history (
  id bigint generated by default as identity primary key,
  user_id text not null,
  image_filename text,
  image_bytes text,
  predicted_class text,
  confidence double precision,
  all_scores text,
  created_at timestamptz default now()
);
```

### feedback
```sql
create table public.feedback (
  id bigint generated by default as identity primary key,
  user_id text not null,
  message text,
  rating integer,
  created_at timestamptz default now()
);
```

## Vercel deployment
This project includes a Vercel-compatible entry point in [api/index.py](api/index.py) and config in [vercel.json](vercel.json).

### Required environment variables in Vercel
- SECRET_KEY
- SUPABASE_URL
- SUPABASE_KEY
- SUPABASE_SERVICE_ROLE_KEY

### Deploy steps
1. Push the repo to GitHub.
2. Import the project in Vercel.
3. Set the project root correctly.
4. Deploy.

## Notes
- For serverless hosting, keep the app lightweight and ensure the model fallback path is available.
- The app uses Supabase for persistence instead of a local database.
